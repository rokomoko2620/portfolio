01

VR もぐらたたき

Unity UdonSharp VRChat マルチプレイヤー
開発期間：2025年8月 オープンキャンパスの研究室の出し物
Overview

VRChatで遊べるマルチプレイヤー対戦ゲームです。モグラ役のプレイヤーが穴から頭を出してゴールを目指し、 ハンマー役のプレイヤーがそれを妨げる対戦ゲームです。マスタープレイヤーが開始ボタンを押すと全員のタイマーが開始され、 地上に留まることでタイマーが減少していき、タイマーが0になった時にクリアとなります。

スクリプト構成
ファイル名役割
MoleGameManager.csゲーム全体の管理（タイマー・同期・テレポート・開始/終了）
HammerHitDetector.csハンマーがプレイヤーに当たったことを検出し通知
HammerFollower.csハンマーオブジェクトをVRコントローラーの手に追従させる
SimpleButtonHandler.cs参加ボタン・開始ボタンのタッチ操作を処理
技術的なポイント

ネットワーク同期には [UdonSynced] を使用し、 マスターが権限を持ちプレイヤー間でゲーム状態を同期しています。 各プレイヤーは自身のみテレポート可能なため、同期されたクリア情報を元に本人側で実行し、 フラグによって重複テレポートを防止しています。

ハンマーの連続ヒットはクールダウン機能で防止され、ヒット時にパーティクルエフェクトやサウンドも再生されます。 最大8人までの参加者を配列で管理し、各プレイヤーのタイマーやゲーム状態を保持する仕組みになっています。

02

窓ツールキット

Unity UdonSharp VRChat シューティング
開発期間：2025年9月 〜 12月 チーム開発（2人）・研究室プロジェクト
Overview

VRChatで現実世界とVR空間を繋ぐシューティングゲームシステムです。 VRプレイヤーがモニターを通して現実のプレイヤーを見ながら撃ち合うことができます。 モニターキャリブレーション・銃システム・レイキャスト射線・スコア管理の機能を提供しています。

スクリプト構成
スクリプト説明
HandController.csVRコントローラーの位置同期、銃の追従・発射
MadoCalibrator.csモニターキャリブレーション、カメラ手動調整
MadoOwnerManager.cs複数モニターの管理、キャリブレーション開始
RayGenerator.cs射線生成、ヒット判定
PlayerTracker.csプレイヤー追跡ヒットボックス
ScoreManager.csスコア管理、ネットワーク同期
技術的なポイント

キャリブレーションでは、VRコントローラーで4隅をポイントし、モニターの位置とVR内のモニターを自動で同期させます。 弾がモニターを通過すると射線を生成し、プレイヤーのヒットボックスを検出する仕組みになっています。

手の位置と回転は UdonSynced + RequestSerialization で同期し、 スコアは SendCustomNetworkEvent で全プレイヤーに通知されます。 射線はローカルのみで、撃った人だけ見える仕様です。

03

CONNECT

PHP MySQL JavaScript Leaflet
開発期間：2025年12月 〜 2026年1月 授業課題・個人再実装
Overview

「〜旅の出会いを、もっと特別に〜」というコンセプトの旅行計画マッチングサイトです。 授業課題でチームで開発した成果物を、自分が担当したコードのみで再実装したプロジェクトです。 OpenStreetMapによる地図上での場所選択や、18種類の旅行目的による検索フィルタリング、 自動マッチング機能などを実装しています。

主な機能
機能説明
旅行計画の作成地図での場所選択・目的やりたいこと（18種類）の設定・写真アップロード
検索・フィルタリング都道府県・旅行目的・やりたいこと・投稿者属性で絞り込み
自動マッチング場所・目的・やりたいことの重要度を考慮した相性スコア計算
コミュニケーションいいね・コメント・1対1チャット・通知バッジ
仮想旅10問の質問で旅の価値観チェック・相性度の自動計算
技術スタック

バックエンドはPHP 7.x、データベースはMySQL（MariaDB）で構築しています。 フロントエンドにはHTML5・CSS3・JavaScriptを使用し、地図機能はLeaflet＋OpenStreetMapで無料で実現しています。

04

線香花火 3D シミュレーション

HTML JavaScript Three.js WebGL / GLSL
開発期間：2025年12月 授業課題
Overview

Three.jsを使用した線香花火のWebGL 3Dシミュレーションです。 線香花火の4段階（蕾→牡丹→松葉→散り菊）の燃え方を再現しており、 3種類の火花タイプを切り替えて楽しめます。360度自由にカメラを回転して鑑賞できます。

火花タイプ
タイプ特徴
タイプ 1枝分かれする火花。液滴が破裂するように最大10回まで分岐
タイプ 2シェーダー火花。GPU計算による滑らかな揺らぎと軌跡
タイプ 3流星タイプ。長い尾を引く火花
技術的なポイント

火花の描画には LineSegments と Points を組み合わせて使用しています。 軌跡はLineSegmentsで、先端はPointsで描画し、リアルな火花の動きを再現しました。

物理シミュレーションでは重力と空気抵抗を考慮した位置更新を行い、 火球シェーダーでは頂点シェーダーで表面の揺らぎを、フラグメントシェーダーでフレネル効果による発光表現を実装しています。 Bloomエフェクトによるポストプロセッシングでさらにリアルな発光を強調しています。
